The following is a set of guidelines for contributing to SafeScore. These are mostly guidelines, not rules. Use your best judgment, and feel free to propose changes to this document in a pull request.

## How Can I Contribute?

### Reporting Bugs
*   Check the [Issues](https://github.com/devtofunmi/safescore/issues) to see if the bug has already been reported.
*   If not, open a new issue. Include a clear title and description, as many relevant details as possible, and a code sample or an executable test case demonstrating the expected behavior that is not occurring.

### Suggesting Enhancements
*   Open an [Issue](https://github.com/devtofunmi/safescore/issues) to discuss the enhancement.
*   Provide a clear and detailed description of the proposed enhancement and its benefits.

### Pull Requests
*   Fill in the required template.
*   Keep your PR small and focused on a single change.
*   Include screenshots and/or videos for UI changes.
*   Ensure the code passes all linting and build checks.

## Style Guide

### Git Commit Messages
*   Use the present tense ("Add feature" not "Added feature")
*   Use the imperative mood ("Move cursor to..." not "Moves cursor to...")
*   Limit the first line to 72 characters or less

### Code Style
*   We use ESLint and Prettier for code formatting.
*   Run `npm run lint` before submitting a PR.

## Questions?
If you have any questions, feel free to open an issue or reach out to the project maintainers.
